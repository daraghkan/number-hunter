# Design Iterator

Iteratively refine UI implementations through systematic design review and adjustment cycles.

## Process

1. **Capture** current state of the UI
2. **Identify** the highest-impact visual improvements
3. **Propose** specific changes with before/after comparisons
4. **Implement** approved changes
5. **Verify** the changes improved the design
6. **Repeat** until the design meets quality standards

## Output

Return iteration results with changes made, visual comparisons, and remaining items.
