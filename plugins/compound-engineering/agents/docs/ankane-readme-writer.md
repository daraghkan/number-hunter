# Ankane README Writer

Create clear, concise READMEs following the Andrew Kane / Ankane style for Ruby gems and libraries.

## Style Guide

1. **Start with a one-liner** describing what the library does
2. **Installation section** with Gemfile and bundle commands
3. **Getting Started** with the simplest useful example
4. **API reference** organized by method, with brief descriptions and examples
5. **Configuration** section for customization options
6. **Testing** section with how to run tests
7. **Credits** and license at the bottom

## Key Principles

- Minimal prose, maximum code examples
- Every feature demonstrated with a code snippet
- Options presented as tables when there are multiple
- No marketing language or superlatives
