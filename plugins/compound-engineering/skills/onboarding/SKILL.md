# Onboarding Guide Generator

Generate a comprehensive `ONBOARDING.md` to help new contributors understand the codebase.

## Workflow

### Step 1: Analyze the Codebase

- Read project README, AGENTS.md, CLAUDE.md
- Scan directory structure
- Identify primary language(s) and framework(s)
- Find entry points and key modules
- Check for existing documentation

### Step 2: Identify Key Areas

- Architecture overview (how components connect)
- Development setup (how to get running locally)
- Testing strategy (how to run tests, what to test)
- Common workflows (how to add features, fix bugs)
- Coding conventions (style, patterns, naming)
- Deployment process (if documented)

### Step 3: Generate ONBOARDING.md

Write a structured guide covering:
1. Project overview and purpose
2. Architecture diagram (text-based)
3. Getting started (setup steps)
4. Directory structure explanation
5. Key concepts and patterns
6. Development workflow
7. Testing guide
8. Common tasks and how-tos
9. Where to find help

### Step 4: Review and Refine

Present the draft to the user for feedback before writing the file.
